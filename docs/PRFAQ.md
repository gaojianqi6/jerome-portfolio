# Jerome Portfolio PRFAQ

## Press Release

**标题：Jerome Gao 发布个人双语 Portfolio，用高级设计与清晰前端结构呈现难以被简历看见的全栈能力**

**副标题：不是在线简历，也不是简单项目列表，而是一个用设计、信息架构和案例叙事建立信任的个人产品系统。**

**Auckland, New Zealand - 2026 年 7 月 17 日** - Jerome Gao 今日发布个人双语 Portfolio 网站，面向新西兰及远程市场的招聘经理、技术负责人、工程团队和合作伙伴，集中呈现其 10+ 年全栈工程经验、React / Next.js / TypeScript 前端工程能力、React Native 跨端能力、后端架构能力，以及从 0 到 1 交付产品的真实项目证据。

很多工程师的个人网站只回答了“我会什么技术”，却没有回答“我能解决什么问题、如何思考产品、如何把复杂系统做成用户能用的体验”。全栈能力尤其难展示：后端架构、数据库设计、测试、部署、业务建模通常隐藏在界面背后。Jerome Portfolio 的目标不是简单“展示前端能力”，而是用优秀的视觉设计、清晰的信息架构、可维护的前端结构和深度 case study，把这些隐藏能力变成访问者可以理解、可以验证、可以讨论的证据。

网站首屏会直接呈现 Jerome 的市场定位：Senior Full Stack Engineer with product-minded frontend craft。页面不把“多年经验”当作唯一卖点，而是把能力证据产品化：Carsome 的高性能响应式官网与 CMS 微前端、Te Kemu Arapū 的 React Native 实时多人游戏、RateEverything 的 Next.js / NestJS / FastAPI 全栈评分平台、Osprey Pulse 与 Mealway 这类持续迭代的跨端产品，都被重构为可浏览的 case study。每个 case study 不只列技术栈，还回答业务背景、用户场景、关键难点、我的角色、设计取舍、前端结构、后端边界、工程质量和最终结果。

“我希望别人看到的不只是我做过什么，而是我如何把不清晰的问题变成可交付的产品。”Jerome Gao 表示，“全栈能力很难只靠页面截图展示，所以 Portfolio 本身必须有好的设计判断和前端结构：信息架构、响应式布局、可访问性、交互动效、性能、内容模型、组件组织和双语表达，都必须体现在这个网站里。”

Jerome Portfolio 会采用完整双语内容策略，英文和中文都作为一等内容，而不是简单机器翻译。英文面向新西兰本地招聘和国际远程机会，中文展示 Jerome 的双语沟通能力、跨文化背景和中文技术表达能力。网站将支持桌面和移动端访问，重点优化招聘方常见的快速浏览路径：打开网站、确认定位、查看代表项目、下载 CV、通过 email / LinkedIn / GitHub 联系。对于技术面试官，则提供更深的工程路径：打开项目、查看架构和代码亮点、理解性能优化、测试覆盖、部署方式与技术决策。

这个 Portfolio 的核心不是视觉炫技，而是信任建立。视觉上会保持高级、克制、工程感强，避免模板化的个人主页；交互上会通过项目筛选、案例切换、技术标签、可展开架构说明、微交互动效和真实链接，让访问者感受到 Jerome 对前端体验和产品细节的控制力。对于没有时间阅读全部内容的人，网站会提供清晰的“3 分钟路径”；对于愿意深入的人，网站会提供“15 分钟路径”，让其完整理解 Jerome 的技术宽度、产品判断和工程深度。

首个版本将包括双语首页、关于我、项目案例、技能矩阵、工作经历、直接联系方式和 CV 下载。不会加入 contact form，因为 Jerome 已经有清晰 email 联系方式，表单会增加维护成本和垃圾信息风险。后续版本会加入交互式项目 demo、可视化架构图、博客/工程笔记、招聘岗位定制入口，以及基于访问场景的内容推荐。成功标准不是页面数量，而是访问者能否在最短时间内形成三个判断：这个人可信吗、能力是否匹配、是否值得联系。

Jerome Portfolio 将成为 Jerome 的长期职业资产：它持续吸收新的项目、文章、实验和产品思考，逐步从“求职材料”升级为“个人工程品牌”。它让更多人看见 Jerome，也让 Jerome 的全栈能力不再只被简历描述，而是通过设计质量、前端结构、项目叙事和双语表达被网站本身证明。

## FAQ

### 1. 这个 Portfolio 的主要用户是谁？

第一优先级是招聘经理、技术负责人和面试官；第二优先级是工程同行、潜在合作伙伴、客户和社区成员。不同用户的时间预算不同，因此网站必须同时支持快速判断和深度验证。

### 2. 为什么不只做一个在线简历？

在线简历只能陈述经历，Portfolio 要展示判断力和执行力。全栈能力常常藏在系统内部，所以网站需要通过案例结构、架构说明、内容层级和真实链接把它翻译出来。同时，网站本身的视觉、组件、响应式、加载状态、可访问性、性能与内容组织会直接暴露 Jerome 的前端审美和工程水平。

### 3. 网站最重要的一句话是什么？

Jerome 是一位拥有 10+ 年经验、具备强产品判断和前端结构能力、可以端到端交付产品的 Senior Full Stack Engineer。

### 4. 首页应该让访问者先看到什么？

先看到定位、可信证据和行动入口。首屏需要包含姓名、角色定位、核心价值、关键技术、所在地、双语切换、代表项目入口、下载 CV 和 email / LinkedIn / GitHub 直达入口。不要先讲长故事，也不要放 contact form。

### 5. 如何让全栈能力被看见？

通过真实项目 case study、系统边界图、前后端协作路径、数据模型、API 设计说明、测试策略、部署方式、代码链接和 live demo 共同证明。全栈能力不是靠“我会后端”这句话证明，而是靠“我如何把产品问题拆成界面、状态、接口、数据库、测试和部署”来证明。

### 6. 为什么仍然要强调设计和前端结构？

因为 Portfolio 是访问者唯一能立即体验到的作品。好的设计能力能让复杂经历变清晰，好的前端结构能让网站可维护、可扩展、可双语化，也能间接证明 Jerome 具备产品化思维，而不是只会堆技术栈。

### 7. 是否需要博客？

首版不强制。博客可以作为后续增长模块，用来展示思考深度，例如 React 架构、Next.js 性能、跨端 monorepo、GraphQL、测试和 AI 产品实践。首版先保证核心转化路径稳定。

### 8. 是否需要双语？

需要，而且首版就应该有。Jerome 本人是中英双语，Portfolio 应该把这件事展示出来。英文面向新西兰和国际市场，中文展示跨文化沟通能力、中文技术表达和个人真实背景。双语内容必须结构化维护，不能只是页面上零散切换文本。

### 9. 是否需要 contact form？

不需要。首版直接提供 email、LinkedIn、GitHub 和 CV 下载即可。Contact form 会引入 spam、验证、邮件服务、错误处理和隐私说明，收益不高。对招聘和合作场景来说，直接 email 更自然、更可信。

### 10. 访问者最可能的疑虑是什么？

疑虑包括：项目是否真实、本人贡献是否明确、技术是否停留在表面、是否能适应本地团队沟通、是否只会前端不懂系统、是否有维护质量意识。因此每个项目必须明确“我的角色”和“可验证证据”。

### 11. 首版成功如何衡量？

定量指标包括 email 点击率、CV 下载量、项目页停留时长、深度滚动率、GitHub / LinkedIn 点击率、中英文切换使用情况；定性指标包括招聘方能否复述 Jerome 的定位、是否主动提到某个项目、是否将网站作为面试讨论材料。

### 12. 最大风险是什么？

最大风险是做成“漂亮但空”的页面。解决方法是先确定内容证据链，再做视觉和动效；每个模块都必须服务于一个判断：让访问者更快、更确定地相信 Jerome 适合他们的岗位或合作场景。
